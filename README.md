# AI Safety Scanning System (`safescan.js`)

This document explains how to use and configure the AI Safety Scanning System, implemented primarily through the `src/utils/safescan.js` utility. This system helps prevent content policy violations by analyzing user-provided images and prompts before sending requests to external AI generation services like FAL.ai.

## üéØ Purpose

The primary goal of the `safescan.js` utility is to:
- **Detect potential content policy violations** in user inputs (images and text prompts).
- **Warn users** about flagged content, indicating that generating such requests may lead to failed results and non-refundable token usage.
- **Provide actionable suggestions** for users to modify their content.
- **Prevent unnecessary token consumption** by blocking or warning against problematic generations before they are sent to costly external APIs.

## ‚öôÔ∏è How It Works

The system operates in two main phases:

1.  **Pre-submission Analysis**: When a user initiates a generation (e.g., clicks "Generate"), the system intercepts the request and performs an AI-powered analysis of the input image and/or prompt text.
    -   **Image Analysis**: Uses OpenAI Vision to analyze the visual content for nudity, violence, hate speech, etc.
    -   **Prompt Analysis**: Uses OpenAI to analyze the text prompt for inappropriate language or requests. It also includes a quick keyword check for common problematic terms.
2.  **User Notification**:
    -   If the analysis deems the content **safe**, the generation request proceeds directly to the external AI service.
    -   If the analysis produces a **negative response** (i.e., content is flagged), a `SafetyWarningModal` is displayed to the user. This modal:
        -   Clearly indicates what was flagged (image, prompt, or both).
        -   Explains that tokens will **not be refunded** if the generation fails due to content policy constraints.
        -   Offers the user the choice to "Continue Anyway" (bypassing the warning) or "Modify Content" (allowing them to adjust their input).
    -   If the user chooses to "Continue Anyway", the request is sent to the external AI service, but the system logs that the user proceeded despite the warning.

## üõ†Ô∏è Configuration (`src/utils/safescan.js`)

The behavior of the safety scanner is controlled by the `safeScanConfig` object within `src/utils/safescan.js`.

```javascript
export const safeScanConfig = {
  enabled: true, // Set to false to disable safety scanning globally
  
  // Sensitivity levels (1-5, where 5 is most strict)
  imageSensitivity: 3, // Adjust based on desired strictness for images
  promptSensitivity: 3, // Adjust based on desired strictness for prompts
  
  // Which tools to scan (set to false to disable for specific tools)
  // The key should match the `toolType` passed to performSafetyAnalysis
  toolsToScan: {
    'fal_wan_v22_video2video': true, // Example: Enable scanning for WAN v2.2 Video2Video
    'fal_wan_pro': true,
    'fal_kling_pro': true,
    'fal_minimax_hailuo': true,
    'fal_veo2': true,
    'fal_ltxv': true,
    'fal_seedance_pro': true,
    'fal_wan_v22_a14b': true,
    'fal_flux_kontext': true,
    'fal_flux_kontext_lora': true,
    'fal_flux_redux': true,
    'fal_hidream_i1': true,
    'ai_scene_gen': true,
    'fal_omnihuman': true,
    'fal_cassetteai_music': false, // Example: Disable scanning for music generation
    'fal_mmaudio_v2': false,
```

## üöÄ Integrating with AI Tools

To integrate the safety scanning system into an AI generation tool, follow these steps:

1.  **Import necessary utilities**:
    ```javascript
    import { performSafetyAnalysis, shouldShowWarning, getSafetyWarningMessage, logSafetyAnalysis } from '../../utils/safescan';
    import SafetyWarningModal from '../../components/SafetyWarningModal';
    ```

2.  **Manage state for the warning modal**:
    ```javascript
    const [showSafetyWarning, setShowSafetyWarning] = useState(false);
    const [safetyWarningData, setSafetyWarningData] = useState(null);
    const [bypassSafetyCheck, setBypassSafetyCheck] = useState(false); // New state for bypass
    ```

3.  **Modify your `handleGenerate` function**:
    Wrap your existing generation logic with the safety analysis.

    ```javascript
    const handleGenerate = async () => {
      // ... (your existing input validations) ...

      // --- SAFETY SCAN INTEGRATION ---
      // Only perform safety check if not bypassed
      if (!bypassSafetyCheck) {
        const analysisResult = await performSafetyAnalysis(
          config.imageUrl, // Pass image URL if applicable, otherwise null
          config.prompt,   // Pass prompt text
          'your_tool_type_here' // IMPORTANT: Use the toolType string from safescan.js config
        );

        logSafetyAnalysis(analysisResult, 'pre_generation_check'); // Log the analysis result

        if (shouldShowWarning(analysisResult)) {
          setSafetyWarningData(getSafetyWarningMessage(analysisResult));
          setShowSafetyWarning(true);
          return; // Stop generation, wait for user decision
        }
      }

      // Reset bypass flag after the check (or if no check was needed)
      setBypassSafetyCheck(false);

      // ... (your existing token check and generation logic) ...
      // This part will only run if the safety check passes or is bypassed.
    };
    ```

4.  **Add the `SafetyWarningModal` component to your JSX**:
    Place this component near the end of your tool's main JSX return, typically before the closing `</div>` of your main content wrapper.

    ```javascript
    {/* Safety Warning Modal */}
    <SafetyWarningModal
      isOpen={showSafetyWarning}
      onClose={() => setShowSafetyWarning(false)} // User chose to modify
      onContinue={() => {
        setBypassSafetyCheck(true); // Set flag to bypass next check
        setShowSafetyWarning(false); // Close modal
        handleGenerate(); // Re-trigger generation
      }}
      onModify={() => {
        setShowSafetyWarning(false); // Close modal
        // User can now modify prompt/image, and it will be re-checked on next generate click
      }}
      warningData={safetyWarningData}
      toolName="Your Tool Name" // Display name for the tool in the modal
    />
    ```

## ‚ûï Adding New AI Tools

To enable safety scanning for a new AI tool:

1.  **Define a `toolType` string**: Choose a unique, descriptive string for your new tool (e.g., `'my_new_image_tool'`, `'video_editor_pro'`).

2.  **Update `src/utils/safescan.js`**:
    -   Open `src/utils/safescan.js`.
    -   Locate the `toolsToScan` object within `safeScanConfig`.
    -   Add your new `toolType` as a key and set its value to `true` to enable scanning, or `false` to disable it.
        ```javascript
        export const safeScanConfig = {
          // ... other configs ...
          toolsToScan: {
            // ... existing tools ...
            'my_new_image_tool': true, // Enable scanning for your new tool
            'video_editor_pro': true,
          },
          // ... rest of config ...
        };
        ```

3.  **Integrate into your new AI tool's component**:
    -   Follow the "Integrating with AI Tools" steps above.
    -   Ensure that the `toolType` argument passed to `performSafetyAnalysis` exactly matches the string you defined in `safeScanConfig.toolsToScan`.
        ```javascript
        const analysisResult = await performSafetyAnalysis(
          config.imageUrl,
          config.prompt,
          'my_new_image_tool' // This must match the key in safescan.js
        );
        ```

By following these steps, any new AI tool can leverage the centralized safety scanning system, ensuring consistent content moderation across your platform.
